# Durable Improvements

반복되는 에이전트 process 교정과 재사용 가능한 개선 규칙을 최소 단위로 기록한다.
프로젝트 사실, endpoint 목록, private 운영 절차는 이곳에 중복하지 않는다.

## When To Record

- 같은 사용자 교정 또는 process gap이 두 번 이상 발생했다.
- 이번 작업에서 확인한 규칙이 다음 작업의 오류·불필요한 탐색을 줄인다.
- rule을 root guide, task/workflow guide, decision note, reusable skill 중 어디에
  승격할지 판단할 근거가 필요하다.

## Entry Format

새 entry는 제목·발견일·문제·규칙·적용 범위·승격 대상·상태를 짧게 적는다. 공개 저장소
안전 규칙을 지키고, sensitive operational context는 private worklog에만 둔다.

```md
## YYYY-MM-DD - 짧은 개선 제목

문제: <반복된 문제>
규칙: <앞으로 따를 행동>
적용 범위: <어떤 작업에 적용하는가>
승격 대상: <root/task/workflow/decision/skill 또는 없음>
상태: active | promoted | superseded
```

## Current Entries

아직 기록된 항목이 없다.
